1. IDENTITY & PRIMARY GOAL
Project Name: Lumière Bistro & Experiences
Primary Function: High-end restaurant discovery and experience booking platform with a secure Admin Portal for reservation management.
Development History: Originally scaffolded in v0; migrated to GitHub for cross-platform development (Bolt, Lovable, etc.).

2. THE TECH STACK (DNA)
Framework: Next.js 15 (App Router).
Styling: Tailwind CSS 4.0.
UI Components: Shadcn/UI (Radix UI primitives).
Animations: Framer Motion (integrated for smooth section transitions).
Icons: Lucide React.
Data/Auth: Supabase (PostgreSQL) with @supabase/ssr.
Email: Nodemailer (for automated booking confirmations).

7. CORE LOGIC FLOWS
Authentication: Managed in lib/supabase/staff-auth.ts. Uses bcryptjs for hashing and staff_session cookies for persistence.
Middleware Security: middleware.ts uses lib/supabase/proxy.ts to intercept requests. Any path matching /admin is redirected to /login if a valid session is not found.
Booking Calculations: lib/booking-utils.ts fetches restaurant_config and bookings for a specific date to calculate "Remaining Covers" before allowing a new reservation.
Email Notifications: lib/emails.ts triggers Nodemailer to send HTML-formatted emails to both the customer and the staff_notification_email upon successful booking.
